# Start Project Wizard

## Purpose

This file defines the structure of the Viral Prints Start Project wizard.

The goal is to make it easy for customers to submit the right information without overwhelming them.

The wizard should be embedded on the website using Jotform or a similar tool.

---

## Step 1: Customer and Business Information

Collect:
- full name
- business name
- email
- phone
- address
- website or social handle
- preferred contact method

---

## Step 2: Project Location

Collect:
- indoor or outdoor
- project address
- wall or surface type
- surface condition
- access restrictions if any
- business hours
- before-hours or after-hours preference if needed

---

## Step 3: Wall Details

Collect:
- approximate width
- approximate height
- estimated viewing distance if useful
- upload wall photos
- upload additional space photos if needed

---

## Step 4: Design Path

Customer chooses one:

### Option A: Upload Their Own Design
Collect:
- artwork file upload
- note about ownership or usage rights
- optional notes about edits needed

### Option B: Choose From Curated Catalog
Collect:
- industry
- style
- mood
- color direction
- image category
- inspiration preferences

### Option C: Request Custom Design
Collect:
- business type
- target audience
- visual style
- color direction
- business goal for the mural or print
- uploaded inspiration
- uploaded brand assets
- notes

---

## Step 5: Package Selection

Collect:
- Core
- Signature
- Vision
- size tier
- add-ons if applicable

---

## Step 6: Scheduling

Collect:
- 2 to 3 preferred install windows
- timeline goal
- flexibility
- business disruption concerns

Note:
The website should communicate that the team confirms the final install window within 24 to 48 hours.

---

## Step 7: Final Notes

Collect:
- special instructions
- budget notes if useful
- target launch or event date
- anything else the team should know

---

## Form Design Notes

The wizard should:
- be easy to complete
- avoid long intimidating sections
- only ask required questions when needed
- allow uploads
- clearly explain next steps after submission

---

## Submission Outcome

After submission:
- create or update project record in Airtable
- assign project ID
- notify internal team
- move lead into quote or review workflow
