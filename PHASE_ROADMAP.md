# Phase Roadmap

## Purpose

This file explains what Viral Prints should build now, what should be prepared for later, and what should remain future-facing until the business is ready.

The goal is to avoid overbuilding too early while still creating a system that can grow without a full rebuild.

---

## Phase 1 MVP

### Goal
Launch a clean, conversion-friendly website and backend process that makes it easy for customers to start a project.

### Active Features
- marketing website
- Home page
- How It Works page
- Packages page
- Gallery page
- FAQ page
- Contact page
- Start Project CTA on every page
- embedded Jotform intake flow
- Airtable backend tracking
- Stripe deposit and final payment workflow
- proof delivery by email
- approval or revision form
- referral QR generation logic
- project ID tracking

### Phase 1 Priorities
- clear 4-step buying process
- package structure
- easy intake flow
- clean quote and follow-up workflow
- organized backend tracking
- no-code or low-code tools

### Build Rule
Phase 1 should be fast, low-cost, and good enough to launch.

---

## Phase 2

### Goal
Improve customer visibility and operational efficiency.

### Planned Features
- client portal login
- status tracking
- uploads center
- proofs center
- approval dashboard
- payment history
- stalled project reminders
- stronger admin dashboard
- stronger internal workflow tools

### Build Rule
Do not fully build these yet, but Phase 1 should not block them later.

---

## Phase 3

### Goal
Increase buying confidence through better visualization.

### Planned Features
- wall photo mockup preview
- design overlay onto customer wall image
- browser-based preview tools
- later AR preview direction
- stronger scope and sizing guidance

### Build Rule
Research and structure for later, but do not require this for launch.

---

## Phase 4

### Goal
Prepare for scale and future multi-location or franchise operations.

### Planned Features
- multi-location support
- roles and permissions
- crew scheduling
- territory logic
- regional analytics
- franchise-ready workflows

### Build Rule
This should influence backend thinking, but none of these features need to be active in early build stages.

---

## What Must Be Built Now vs Later

### Build Now
- website pages
- Start Project wizard
- package structure
- quote intake flow
- Airtable schema
- proof approval process
- Stripe payment process

### Prepare For Later
- client portal
- mockup preview
- automation improvements
- referral analytics
- role-based admin tools

### Do Later
- AR tools
- franchise systems
- crew management
- territory controls
